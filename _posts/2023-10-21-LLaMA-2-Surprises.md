The last post was a deep dive into the guts of Llama 2, so I want to zoom out and go over a 
few of the highlights.  The full adventure is [linked here](https://landrewwray.github.io/2023/10/19/Inside-LLaMA-2.html), and I'll revisit some of it in future posts that may also be linked on this page.  Llama 2 is an open source cousin of ChatGPT that follows the same ["decoder-only" development path](https://www.interconnects.ai/p/llm-development-paths).

The high points for me were:

### 1. Internal structure of the model:

From the outside, large language models look like very deep neural networks composed of sequential 'transformer' layers that all share the same design (32 layes in Llama 2 7B!)  This repetitive architecture makes it hard to develop intuition for how the model 'thinks', or where one should direct effort when fine-tuning the models for specific applications.

However, the parameters that go into these layers look very different in different parts of the network.  From what I can see, the model seems to act in four overlapping phases:

1. **Translation** (layers 0-1): The input to the first transformer layer is a set of 4096-long state vectors, each of which represents one of the tokens (word-parts) in the text the model is working on.  The first two layers radically transform these state vectors.  Some of the operations they appear to perform include: (1) Translating multi-token words into single vectors.  Llama only recognizes 32,000 tokens, but there are ~200,000 commonly used words in the English language, so many words have multi-token repsresentations.  (2) Tokens belonging to *different* words are also translated into vector representations that encode their mutual relationships association.  For example, when the model reads, "I like the red ball", the first two transformer layers cause the "ball" vector to contain [representations of the words "I" and "red"](https://landrewwray.github.io/2023/10/19/Inside-LLaMA-2.html#2b-internal-dictionaries-of-an-llm).  (3) The first layers also identify certain kinds of boundaries, such as the [opening and closing of quotes or parenthetical notes](https://landrewwray.github.io/2023/10/19/2023-10-19-Inside-LLaMA-2.md#3-what-words-do-the-attention-heads-look-for).

2. **Free association** (layers 0-16): The model doesn't just translate collections of words (or tokens) into combined vector representations -- it also calls up a wide range of memories connected with the content it's addressing.  A <a href = "https://arxiv.org/abs/2310.02207" target = "_blank" rel = "noreferrer noopener">recent paper</a> found that a great deal of factual information related to the content of the prompt tends to be called up within the first 25-50% of the model.  For example, if a place name occurs in the prompt, then by the middle of the model (say, layer #12) the corresponding state vector is likely to contain latitude and longitude information.  This kind of free association is very similar to the primary function of many "encoder" neural networks - something that I'll get back to in a later post.

3. **Contextual reasoning** (layers 12-30): Contextual logic is visible in the second half of the model.  For example, if the prompt reads "I like to play", the first half of the model will contain next-word ideas like "ground" (playground) or "test" (play test) that have meaning when combined with the word "play", but aren't informed by the broader context.  Starting in the second half of the model, one finds next word ideas that are [increasingly well informed](https://landrewwray.github.io/2023/10/19/Inside-LLaMA-2.html#2b-internal-dictionaries-of-an-llm).

4. **Purging** (layers 30-31): The last couple of layers have a vital role: they purge the state vectors of all content that is not a next word prediction.  All the word associations and recalled memories need to disappear, so that they don't pollute the model output. This specialization runs somewhat counter to the prevailing wisdom that the final layers of a neural network deal with edge cases, and are good targets for specialized fine tuning.

### 2. The internal states are surprisingly decipherable.

There are several [matrices that I'm calling 'dictionaries'](https://landrewwray.github.io/2023/10/19/Inside-LLaMA-2.html#2b-internal-dictionaries-of-an-llm) that map between meanings (or word tokens) and the internal state vectors that are created during text generation.  The model can remember several words from each dictionary by adding these state vectors together.  For example, if a state vector of the model is created by adding four word vectors together (say, **v = v_red + v_apple + v_purple + v_plum**), the model will effectively store an unordered list of those four words ({'red', 'apple', 'purple', 'v_plum'}). If two of the words come from one dictionary (say, **v_red** and **v_apple** in dictionary \#1) while the other two are from another dictionary (**v_purple**, **v_plum** in dictionary \#2), the model will be able to work separately with those word pairs. 

If you know the right dictionaries, you can see some of what the model is thinking as information passes from one transformer layer to the next within the neural network.  In fact, it turns out that the state of the model changes by very little across a single transformer layer - often by just ~5%.

One particularly accessible dictionary is the matrix that's used at the very end of the model to convert state vectors into next token (word-part) predictions.  If we use this output dictionary to interpret state vectors within earlier layers of the model, we can 'read its mind' and see how it tries out different possibilities for the next output token (word-part). For example, if the prompt says you "go to the basketball court to" - the model will begin by coming up with well motivated but incorrect extensions of the text like "oast", as in "toast". After 8 layers we see "to hold", after 12: "to test"; after 16: "to exercise"; after 20: "to ball" and "to play"... until by layer 32, it has a long list of sensible next words, starting with "play", "shoot", "practice", "have", and "throw". There are at least two other dictionaries that can be used in a similar way, one being the input encoding matrix that holds a memory of the original prompt. The third dictionary stores closely associated input words, but can't be very well understood yet, as all I have for it at the moment is a hacked together first draft.

The model actually starts coming up with possible next tokens (word-parts) *even before the first transformer layer!*  The words of the prompt turn into vectors (via the input dictionary) that read as word completions to the output dictionary.  If you turn the word "to" into an input vector and then 
read it with the output dictionary, you get "ast" (as in "toast"), "pping" (as in "topping"), and so on!  The input encoding vector of "play" reads as words like "ground", "offs", and "test" in the output dictionary.

### 3. Large language models as analogue computers:

There were some clues as to how the length of the state vectors helps define a key aspect of [the model's 'cognitive capacity'](https://landrewwray.github.io/2023/10/19/Inside-LLaMA-2.html#6-lessons-for-llm-architecture): how many words it can hold in mind at the same time and think about relationally before some cross a noise threshold and start to become unintelligible. The answer seems to be something like 100 for Llama 2 7B (with 4096 numbers per vector), but even unintelligible words can come back into focus if the model reinforces them.  The same kind of noise will also place bounds on the stability of non-word information, like continuous representations that it seems to have for
<a href = "https://arxiv.org/abs/2310.02207" target = "_blank" rel = "noreferrer noopener">coordinates in time and space</a>.
(Physicist's note: noisy analog systems can be much more stable when they're not quantum computers!!)

### 4. Initial formatting and prompt engineering:

The model can have a very non-intuitive relationship with formatting in the prompts it's given, which stems from the dual identity of tokens.  Tokens are numbers that language models use to represent words or pieces of words.  For example, there model represents the four words "this is a test" as four tokens: 445, 338, 263, 1243.  However, for a transformer-based model like Llama 2 or ChatGPT, tokens also play a role in the assignment of memory.  If we think of the model as a computer, its closest analogue to random access memory is a set of state vectors that are allocated for each token.  For Llama 2 7B, there are thirty-three such vectors per token, each containing 4096 floating point numbers.  When the model is trained on input with fixed formatting such as a "BOS" character at the start of every prompt or a role-defining first sentence, [it can co-opt some of those memory addresses](https://landrewwray.github.io/2023/10/26/Managing-attention.html) for computational purposes that have nothing to do with the explicit meaning of the tokens.
